# DrakSouls 구현 계획

## 현재 완료
- [x] 3인칭 카메라 (마우스 오빗, InputSystem)
- [x] 플레이어 이동 / 회전 (카메라 기준)

---

## Phase 1 — 플레이어 기반 시스템

> 전투 구현 전에 반드시 갖춰야 할 기반 구조

| 순서 | 항목 | 설명 |
|---|---|---|
| 1 | **CharacterController 도입** | `transform.position` 직접 이동 → CharacterController로 교체. 지형 충돌 및 중력 처리 |
| 2 | **StatSystem** | HP, 스태미나 데이터 관리. 전투 / 구르기 / 사망 모두 이 시스템에 의존 |
| 3 | **AnimatorController 연동** | 이동 / 공격 / 피격 / 사망 애니메이션 상태 머신 구성 |
| 4 | **구르기 (Dodge Roll)** | 스태미나 소모 + 무적 프레임. 다크소울 조작감의 핵심 |
| 5 | **락온 (Lock-On)** | 적을 중심으로 카메라 고정, 스트레이핑 이동 지원 |

---

## Phase 2 — 플레이어 전투

| 순서 | 항목 | 설명 |
|---|---|---|
| 6 | **공격 시스템** | 경공격 콤보 (3타), 강공격. 애니메이션 이벤트로 히트박스 활성화 |
| 7 | **히트박스 / 피격 처리** | 무기 콜라이더 On/Off, `IDamageable` 인터페이스로 범용 피격 처리 |
| 8 | **가드 / 패링** | 방패 막기 + 패링 타이밍 판정 (패링 성공 시 적 경직) |
| 9 | **사망 / 부활** | 소울 드롭 위치 저장, 모닥불 기준으로 부활 |

---

## Phase 3 — 적 AI

| 순서 | 항목 | 설명 |
|---|---|---|
| 10 | **Enemy 상태 머신** | Idle → 순찰 → 추적 → 공격 → 피격 → 사망 |
| 11 | **감지 시스템** | 시야각 + 청각 범위로 플레이어 탐지 |
| 12 | **NavMesh 이동** | 지형을 따라 플레이어 추적 |
| 13 | **Enemy 공격 패턴** | 근접 타격, 경직 / 넉백 처리 |

---

## Phase 4 — 맵 & 게임 루프

| 순서 | 항목 | 설명 |
|---|---|---|
| 14 | **레벨 지형** | ProBuilder로 기본 맵 제작 |
| 15 | **체크포인트 (모닥불)** | 부활 지점 저장, HP / 스태미나 회복, 적 리스폰 |
| 16 | **소울 시스템** | 적 처치 시 드롭 → 시체에서 회수 → 사망 시 소실 |
| 17 | **레벨업 / 능력치** | 모닥불에서 소울로 스탯 (HP, 스태미나, 공격력 등) 강화 |

---

## 작업 순서 요약

```
CharacterController 도입
  └─ StatSystem
       └─ AnimatorController 연동
            └─ 구르기
                 └─ 공격 시스템 + 히트박스
                      └─ 락온
                           └─ Enemy AI
                                └─ 맵 + 게임 루프
```
